CSS Animation

"CSS allows animation of HTML elements without using JavaScript or Flash!"(w3schools). The purpose of this style is to let an element change from one style to another. However, to use this type of a style, you must specify some "keyframes" There are nine properties related to this type of CSS, namely:

  • @keyframes
  • animation-name
  • animation-duration
  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-timing-function
  • animation-fill-mode
  • animation

First we will discuss @keyframes rule. This rule allows the animation to gradually change from one particular style to another at particular times. In order to get the animation working, the coder must bind the animation to an element. The main purpose of this rule is to hold whatever styles an element will have at certain times. For example (taken from w3schools) /* The animation code */ @keyframes example { from {background-color: red;} to {background-color: yellow;} } To create the @keyframes with a name, the animation-name property must be used. It specifies the names of one or more @keyframes which describes the animation(s) to apply to the element. The rule itself consists of keyframe sectors that allocates percentages to when the allocation occurs and a another list containing the styles for the @keyframe rule. Percentages can be listed in any order but it's important to note that they will execute in that exact order.
Animation-duration
This property outlines how long an animation should take to complete. If the property isn't defined then the animation will not occur because of the default value which is set to 0. The time may be specified in seconds 9s) or milliseconds (ms).In other words, this property sets the lenght of time to a particular value and that value will determine how long the animation will take to complete one cycle. Just like the animation-name property, you can use percent to add as many style to an element as much as you like. For example (taken from w3schools) 0% {background-color: red;} 25% {background-color: yellow;} 50% {background-color: blue;} 100% {background-color: green;} }


Animation-delay
" As the name suggests, this property specifies how long the animation will delay for the start of the animation. The animation can either start from the beginning, partway through the animation or at a later time. The time selctor specifies the moment the animation is applied to the specific element. Just like animation-duration, time can be specified using either seconds (s) or milliseconds (ms). For this animation, the unit is required. A positive value means that the animation should begin after the specified time has past. If the default time is chosen, that is 0, it indicates that animation will begin as soon as its applied to the element. On the other hand, if it is negative, it means that the animation will begin partway through its cycle.
